# OnTrack - Passenger & Driver Management System

**OnTrack** is a robust Android application designed to streamline the management of transportation services. It provides a centralized platform for administrators to monitor and manage driver and passenger information, ensuring efficient operations and tracking.

## 🚀 Features

- **Admin Authentication**: Secure login and registration system for administrators.
- **Admin Dashboard**: A high-level overview for navigating between passenger and driver management.
- **Driver Management**: Detailed profiles for drivers, including performance metrics and identification.
- **Passenger Management**: Comprehensive details for passengers to ensure safety and service quality.
- **Modern UI**: Built with Material Design principles, featuring Edge-to-Edge support for a premium look and feel.

## 🛠️ Tech Stack

- **Language**: Java
- **Platform**: Android
- **Build System**: Gradle (Kotlin DSL)
- **UI Framework**: Material Design, AndroidX
- **Layouts**: ConstraintLayout for responsive design

## 📂 Project Structure

- `app/src/main/java/com/training/ontrack/`: Contains the Java source code for all activities.
- `app/src/main/res/layout/`: XML layout files defining the application's user interface.
- `app/src/main/AndroidManifest.xml`: Configuration for app components and permissions.

## 📱 Screens

1.  **Welcome Screen (`MainActivity`)**: The entry point of the application.
2.  **Admin Login/Register**: Secure access control for administrators.
3.  **Admin Dashboard (`AdminDetails`)**: Central hub for managing the system.
-   **Passenger Directory**: List of passengers with detailed view options.
-   **Driver Directory**: List of drivers with detailed view options.

## 🛠️ Installation & Setup

1.  Clone the repository:
    ```bash
    git clone https://github.com/AnirbanRay20/OnTrack.git
    ```
2.  Open the project in **Android Studio**.
3.  Sync the project with Gradle files.
4.  Run the app on an emulator or a physical Android device (Minimum SDK: 24).
